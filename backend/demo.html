Food Delivery API Demo

This page demonstrates all backend API endpoints with live testing functionality.

GET /api/restaurants

Retrieve all available restaurants.

GET /api/menu/:restaurant_id

Get menu items for a specific restaurant.

POST /api/register

Register a new user.

POST /api/orders

Place a new order with items.

GET /api/orders/:user_id

Retrieve all orders for a specific user.

PUT /api/orders/:order_id/status

Update order status (UPDATE operation).

PUT /api/menu/:item_id

Update menu item details (UPDATE operation).

DELETE /api/menu/:item_id

Delete a menu item (DELETE operation).

DELETE /api/orders/:order_id

Cancel an order (DELETE operation - soft delete).

GET /api/analytics/dashboard

Get comprehensive analytics and insights (Complex Queries).

SQL Triggers Demo (4 Triggers)

The database has multiple triggers:

  1. increment_order_count: Auto-increments restaurant order_count when order is placed
  2. prevent_restaurant_delete: Prevents deleting restaurants with active orders
  3. update_order_total: Auto-updates order total when items change
  4. log_order_status_change: Logs status changes for audit trail

Test: Place an order, then check restaurants list to see order_count increment!
