Show 4 Triggers (Automatic Actions) (2 minutes)

What is a Trigger?

Simple explanation: "A trigger is like a motion sensor light - it automatically does something when an event happens. In my database, when someone places an order, the restaurant's order count automatically increases."

Show the Triggers in Code:

  1. In schema.sql, scroll to bottom where triggers are
  2. Point to each trigger:

Explain Each Trigger:

Trigger 1 - increment_order_count:
"When a new order is placed, this automatically adds 1 to the restaurant's order count."

Trigger 2 - prevent_restaurant_delete:
"This prevents deleting a restaurant if it has active orders - data safety."

Trigger 3 - update_order_total:
"Automatically recalculates the order total when items are added."

Trigger 4 - log_order_status_change:
"Records every status change for audit trail - like a security log."

Simple explanation: "CRUD stands for Create, Read, Update, Delete - the 4 basic things you can do with data. Like managing your phone contacts - add new contact (Create), view contacts (Read), change a number (Update), remove contact (Delete)."

Stored Procedures & Functions (1 minute)

What are Procedures & Functions?

Stored Procedure: Like a recipe saved in the database - a set of SQL commands that execute together

Function: Like a calculator button - give it input, get a result

Show in Code:

  1. In procedures_functions.sql, scroll to top
  2. Say: "I have 4 stored procedures and 4 functions documented here"
  3. Quickly mention each:

List Them Out:

4 Stored Procedures:

  1. Calculate Restaurant Revenue (with date filtering)
  2. Get Top Selling Items (finds bestsellers)
  3. Update Order Status (with validation)
  4. Get Customer Order History (complete details)

4 Functions:

  1. Calculate User Lifetime Value (total spending)
  2. Get Available Menu Items (only in-stock items)
  3. Calculate Order Item Count (items in cart)
  4. Check Restaurant Availability (has items or not)
Note: SQLite doesn't support stored procedures natively, so these are implemented in the application layer (backend/app.py). This is documented in the SQL file.
